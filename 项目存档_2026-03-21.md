## 4. 当前主要风险
1. 缺少 Git 基线
- 问题: 当前目录非 Git 仓库，无法按 commit 精确回溯。
- 影响: 故障定位、配置追踪、热修复回滚成本高。

2. 部署路径并行（Docker 与 Python 直启混用）
- 问题: 运行方式、依赖来源、配置入口可能不一致。
- 影响: 同一问题在不同部署方式下表现不一致。

3. 默认安全配置风险
- 问题: 配置模板中存在默认口令/默认密钥占位、CORS 默认全开（`*`）。
- 影响: 生产暴露面偏大，存在被误用风险。

4. 自动建表策略在生产可控性不足
- 现状: `backend/main.py` 启动时执行 `Base.metadata.create_all(...)`。
- 影响: 难以替代数据库迁移流程，不利于结构变更审计与灰度。
