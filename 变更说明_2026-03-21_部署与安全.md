# 变更说明（2026-03-21）

## 本次目标
- 建立可回溯的 Git 存档基线
- 统一生产部署主路径为 Docker Compose
- 收紧安全配置（强制非默认密钥、限制 CORS、移除弱口令示例）

## 关键变更
1. 安全配置强约束
- 在 `backend/app/core/config.py` 增加配置校验：
  - `SECRET_KEY` 不能是默认占位值且长度至少 32
  - `DEBUG=False` 时禁止 `CORS_ORIGINS` 包含 `"*"`
- 更新 `backend/.env.example` 与 `backend/.env.server`：
  - 移除弱默认口令/默认密钥
  - 改为明确 CORS 白名单
  - 新增 `ENABLE_BOOTSTRAP_TEST_USER=False`

2. 生产部署路径统一
- 生产主路径固定为 `docker compose -f docker-compose.prod.yml up -d --build`
- `backend/docker-compose.prod.yml` 改为要求显式 `DATABASE_PASSWORD`
- `backend/deploy.sh` 首次部署自动安全初始化：
  - 自动生成随机 `SECRET_KEY`
  - 自动生成随机 `DATABASE_PASSWORD` 并同步 `DATABASE_URL`
- `backend/entrypoint.sh` 默认不创建测试用户，仅在 `ENABLE_BOOTSTRAP_TEST_USER=True` 时执行

3. 弱口令示例清理
- 移除/替换文档与脚本中的 `password123` 示例
- `backend/create_test_user.py` 改为：
  - 优先读取 `TEST_USER_PASSWORD`
  - 未设置时自动生成强随机密码
- 更新 `backend/test_api.py`、`backend/test_main.py` 的示例/测试口令为强口令样式

4. 存档准备
- 新增仓库根目录 `.gitignore`，忽略本地环境文件、构建产物和缓存

## 影响说明
- 若 `SECRET_KEY` 未配置或仍为占位值，服务将拒绝启动（符合预期）
- 若生产环境将 `CORS_ORIGINS` 配为 `"*"`，服务将拒绝启动（符合预期）
- 生产环境默认不会再自动创建固定测试账号

## 回退提示
- 如需临时回退行为，可在 `.env` 中显式设置并确认风险：
  - `ENABLE_BOOTSTRAP_TEST_USER=True`（仅用于排障）
  - 修复完成后应恢复为 `False`
