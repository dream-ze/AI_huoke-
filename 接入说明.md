# browser_collector 接入说明（v2）

## 目标

- 验证链路：本地一键启动服务并导出 Excel
- 主系统接入：AI_huoke 后端通过 HTTP 调用采集服务

## 一键验证（启动服务 + 导出 Excel）

在 PowerShell 执行：

```powershell
powershell -ExecutionPolicy Bypass -File D:\Playwright爬虫\browser_collector\scripts\start_and_export_excel.ps1 -Keyword 贷款 -MaxItems 10 -NeedDetail -NeedComments -TimeoutSec 180
```

执行完成后会在以下目录生成 Excel：

- `D:\Playwright爬虫\exports`

说明：

- 脚本会自动启动服务、调用导出脚本、最后自动停止服务进程
- 如果你只测列表速度，把 `-NeedDetail -NeedComments` 去掉即可

## AI_huoke 主系统接入（主要）

推荐直接复用脚本：

- `browser_collector/scripts/ai_huoke_integration_client.py`

把该文件复制到 AI_huoke 项目后，在业务代码中这样调用：

```python
from ai_huoke_integration_client import BrowserCollectorClient

client = BrowserCollectorClient(base_url="http://127.0.0.1:8005", timeout=240)

# 1) 健康检查
health = client.health()
print(health)

# 2) 采集（v2）
result = client.collect_content(
    keyword="贷款",
    max_items=10,
    need_detail=True,
    need_comments=True,
    dedup=True,
    timeout_sec=180,
)

print(result["count"])
print(result["items"][0]["parse_status"] if result.get("items") else None)

# 3) 单条详情补采（可选）
if result.get("items"):
    one = result["items"][0]
    detail = client.collect_detail(url=one.get("url"))
    print(detail.get("success"), detail.get("source_id"))
```

## v2 请求协议（/api/collect/run）

```json
{
  "platform": "xiaohongshu",
  "keyword": "贷款",
  "max_items": 10,
  "need_detail": true,
  "need_comments": true,
  "dedup": true,
  "timeout_sec": 180
}
```

## v2 返回重点字段

- `count`: 本次返回条数
- `stats.discovered`: 列表发现条数
- `stats.detail_attempted`: 详情补采尝试数
- `stats.detail_success`: 详情补采成功数
- `items[].parse_status`: `list_only | detail_success | detail_failed | parse_failed | risk_blocked`
- `items[].risk_status`: `normal | login_required | captcha | blocked`

## 与 AI_huoke 对接建议

- 只对接 v2 协议，不做旧字段兼容
- 入库唯一键建议：`platform + source_id`
- 对 `detail_failed` / `risk_blocked` 的记录支持后续补采重试
- 前端展示优先使用：`title`、`author_name`、`content_text`、`publish_time`、`like_count`、`comment_count`

## 常见问题

1. 连接失败 `[WinError 10061]`
- 原因：采集服务未启动
- 处理：先启动 uvicorn 或直接使用一键验证脚本

2. 返回有数据但 Excel 未生成
- 原因：`items` 为空时脚本不会导出
- 处理：先检查登录态和关键词命中

3. 小红书风控
- 表现：`parse_status=risk_blocked`
- 处理：检查登录状态，必要时重新执行 `save_login.py`
