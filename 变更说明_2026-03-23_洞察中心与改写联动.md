# 变更说明（2026-03-23）

## 本次目标

1. 完整实现「爆款内容采集分析中心」（以下简称**洞察模块**）——从内容采集到主题沉淀的完整闭环
2. 将洞察库检索结果接入 AI 改写链路，使改写提示词能基于已采集的爆款内容特征进行增强

## 一、洞察模块（新增功能）

### 1.1 设计思路

五层闭环：**采集 → 摄入 → 清洗/去重 → AI 分析 → 检索召回**

- **采集**：支持手工导入单条/批量导入（JSON/CSV）/ 未来扩展官方 API；所有来源合法
- **摄入**：统一六类字段（平台信息 / 作者信息 / 内容信息 / 互动数据 / AI 分析结果 / 风控标记）
- **清洗**：以 `source_url` 为主键去重，重复上报直接跳过并计入 `skipped`
- **AI 分析**：异步调用 Doubao 模型，提取主题、受众、内容结构、钩子、CTA、痛点、亮点、风险提示；分析结果回写至内容条目并聚合到对应主题知识库
- **检索召回**：按平台 + 主题名 + 受众标签筛选，返回结构化特征摘要（不返回原文），供生成模块参考

### 1.2 数据库新增模型

文件：`backend/app/models/models.py`

| 模型 | 用途 | 主要字段 |
|------|------|---------|
| `InsightTopic` | 主题知识库 | `name`(unique)、`platform_focus`、`audience_tags`、`common_titles`、`common_pain_points`、`common_structures`、`common_ctas`、`risk_notes`、`content_count` |
| `InsightAuthorProfile` | 爆款账号档案 | `platform`、`author_name`、`author_platform_id`、`fans_count`、`viral_rate`、`avg_engagement`、`topic_id` FK |
| `InsightContentItem` | 内容条目（六类统一字段） | 互动数据（点赞/收藏/转发/评论/播放）、`engagement_score`、`heat_tier`(viral/hot/warm/normal)、AI 分析 JSON 字段、`risk_flag` |
| `InsightCollectTask` | 采集任务记录 | `collect_mode`（by_account/by_keyword/by_topic/link_import/file_import）、`status`（pending/running/done/failed） |

### 1.4 服务层

文件：`backend/app/services/insight_service.py`（全新文件）

核心方法：

| 方法 | 说明 |
|------|------|
| `_calc_engagement()` | 综合互动分 = 收藏×3 + 评论×2.5 + 转发×2 + 点赞×1 + 播放×0.05 |
| `_calc_heat_tier()` | viral≥500 / hot≥200 / warm≥80 / normal |
| `ingest_item()` | 单条摄入：去重 → 自动关联主题/作者 → 计算热度分 |
| `batch_ingest()` | 批量摄入，返回 `(ok, skipped)` 元组 |
| `analyze_with_ai()` | 调 Doubao API → 提取 9 类分析字段 → 回写条目 → 触发主题知识聚合 |
| `_update_topic_knowledge()` | 聚合主题下 Top-50 已分析内容，更新 `common_titles/pain_points/structures/ctas` |
| `refresh_author_stats()` | 重算作者下所有同平台内容，更新 `viral_rate`、`avg_engagement` |
| `retrieve_for_generation()` | 按平台+主题+受众标签检索，返回结构化摘要（无原文） |

## 二、AI 改写联动洞察库

### 2.1 问题

原有 `rewrite_xiaohongshu` / `rewrite_douyin` / `rewrite_zhihu` 端点使用硬编码英文提示词，与洞察库完全脱离，不知道已采集了哪些爆款内容。

### 2.2 变更明细

#### `backend/app/schemas/schemas.py` — `AIRewriteRequest`

新增两个可选字段：
```python
topic_name: Optional[str] = None          # 关联洞察主题名
audience_tags: List[str] = Field(default_factory=list)  # 受众标签
```

#### `backend/app/services/ai_service.py` — 三个改写方法

- 方法签名统一增加 `insight_ctx: dict | None = None` 参数
- 提示词全部改为**中文**，明确各平台风格要求
- 当 `insight_ctx` 中 `reference_count > 0` 时，自动构建参考信息块注入提示词：
  - 标题写法示例（`title_examples`）
  - 内容结构示例（`structure_examples`）
  - 钩子写法示例（`hook_examples`）
  - 痛点表达示例（`pain_point_examples`）
  - 风格摘要（`style_summary`）
  - 风险提示（`risk_reminder`）
  - 尾部声明：`仅供学习风格，不要复制原文`

#### `backend/app/api/endpoints/ai.py` — 三个改写端点

每个端点在调用 `ai_service.rewrite_XXX()` 之前新增检索步骤：
```python
insight_ctx = None
if request.topic_name:
    insight_ctx = InsightService.retrieve_for_generation(
        db,
        owner_id=current_user["user_id"],
        platform="xiaohongshu",   # 对应平台
        topic_name=request.topic_name,
        audience_tags=request.audience_tags or [],
        limit=5,
    )
```

返回结果新增字段：
- `insight_used`：布尔值，是否实际检索到参考
- `insight_reference_count`：命中的参考条目数

#### `desktop/src/lib/api.ts` — `rewriteContent()`

函数签名新增可选参数：
```typescript
topic_name?: string;
audience_tags?: string[];
```

#### `desktop/src/pages/AIPage.tsx`

- 页面加载时并行拉取内容列表与洞察主题列表（`listInsightTopics()`）
- 表单新增「洞察主题（可选）」下拉选择器，"— 不接入洞察库"为默认项
- 改写完成后在结果区展示：
  - ✅ 已接入洞察库，参考了 N 条采集内容
  - ⚠️ 未检索到匹配参考，已按通用模式改写

## 五、当前待办（下次继续）

### P1 — 数据库迁移

洞察模块新增了 4 张表，生产环境需要执行表结构变更：
- 当前项目使用 `Base.metadata.create_all()`（启动时自动建表）
- 若生产库已有数据，建议改用 Alembic 迁移，避免直接 `create_all` 带来的风险
- 迁移脚本参考：`backend/init_db.py`

### P2 — 批量分析性能

`POST /api/insight/analyze/batch` 最大 50 条，每条独立调用 Doubao API，高并发下可能有限速问题，后续考虑：
- 加入 per-user 速率限制（复用现有 `rate_limit.py`）
- 改为任务队列（Celery / RQ）异步消费

### P3 — 浏览器插件采集

洞察模块目前仅支持手工录入和文件导入，原有浏览器插件（`BrowserPluginCollection`）采集的数据尚未自动流入洞察库。
后续可在 `POST /api/ai/plugin/collect` 端点完成后，自动调用 `InsightService.ingest_item()` 将数据同步入库，实现"插件采集 → 洞察库 → 改写参考"的完全自动化闭环。
