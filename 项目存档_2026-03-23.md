## 3. 洞察模块架构速览

```
前端 InsightPage（4 Tab）
    ↕ HTTP
后端 /api/insight/*（13 端点）
    ↕
InsightService
  ├─ ingest_item()         单条摄入 + 去重（source_url）
  ├─ batch_ingest()        批量摄入（max 200）
  ├─ analyze_with_ai()     AI 深度分析 → 回写字段 → 聚合主题知识
  ├─ _update_topic_knowledge()  Top-50 内容特征聚合
  ├─ refresh_author_stats()    作者爆款率/平均互动重算
  └─ retrieve_for_generation() 按平台+主题+受众检索，返回结构化摘要
    ↕
InsightTopic / InsightContentItem / InsightAuthorProfile（ORM）
```

## 4. AI 改写联动链路

```
用户选择[素材 + 目标平台 + 洞察主题]
    ↓
前端 rewriteContent({ content_id, target_platform, topic_name })
    ↓
ai.py 端点
    ├→ InsightService.retrieve_for_generation()  ← 检索洞察库
    │    返回: title_examples / structure_examples / hook_examples
    │           pain_point_examples / style_summary / risk_reminder
    ↓
ai_service.rewrite_XXX(content, insight_ctx=ctx)
    ├→ 若有参考 → 构建参考信息块注入中文提示词
    └→ 调 Doubao API 生成
    ↓
返回 { rewritten, insight_used, insight_reference_count }
    ↓
前端展示"已接入洞察库 N 条参考"
```

## 6. 当前主要风险与待办

### 风险一：自动建表 vs 迁移（继承自上期）

- 现状：启动时 `create_all`，本次新增 4 张表
- 建议：生产环境引入 Alembic 管理表结构变更

### 风险二：批量 AI 分析无速率保护

- `POST /api/insight/analyze/batch`（max 50）对 Doubao API 无限流
- 建议：接入现有 `rate_limit.py`，或改为任务队列异步消费
