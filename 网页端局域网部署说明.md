# 网页端局域网部署说明

目标：先跑网页版，并让同一局域网内其他电脑可访问。

## 一、后端启动（提供 API）

在 backend 目录：

```powershell
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

默认监听 `0.0.0.0:8000`，局域网其他电脑可访问：

- http://服务器IP:8000/docs

## 二、前端启动（网页）

在 desktop 目录：

```powershell
npm install
npm run dev:web:lan
```

访问地址：

- 本机：http://localhost:5173
- 其他电脑：http://前端机器IP:5173

## 三、跨设备访问关键点

1. 放行防火墙端口：
- TCP 5173（前端）
- TCP 8000（后端）

2. 后端跨域：
- 请在 `.env` 中配置明确白名单，例如 `CORS_ORIGINS=["http://局域网IP:5173"]`
- 生产环境禁止使用 `CORS_ORIGINS=["*"]`

3. API 地址策略：
- 前端默认会自动使用当前访问主机的 `:8000` 作为 API 地址
- 若前后端不在同一台机器，设置 `desktop/.env.local`：

```env
VITE_API_BASE_URL=http://后端机器IP:8000
```

## 四、快速自检

1. 其他电脑能否打开 `http://前端机器IP:5173`
2. 登录接口是否成功（浏览器控制台无 CORS 报错）
3. 能否打开 `http://后端机器IP:8000/docs`

## 五、生产建议（下一步）

1. 使用 Nginx 托管前端 build 静态文件
2. 后端走域名 + HTTPS
3. CORS 改成明确白名单
