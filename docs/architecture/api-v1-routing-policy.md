# API v1 路由治理规范

本规范用于约束后端路由演进，避免旧路径长期残留。

## 1. 路由版本策略

- 业务新增接口统一进入 `/api/v1/*`。
- 非紧急情况不再新增 `/api/*` 非版本化路由。
- 已迁移模块禁止保留双写逻辑。

## 2. 退役策略

- 旧路径一旦完成前后端切换，立即从注册链路移除。
- 兼容层最长保留周期不超过 2 个迭代。
- 退役后必须补充以下验证：
  - 新路径接口可用
  - 旧路径返回 404
  - 前端无旧路径跳转

## 3. 当前状态（2026-03-23）

- 已迁移并启用 v1：
  - `/api/v1/collect/*`
  - `/api/v1/ai/*`
- 已退役旧路径：
  - `/api/collect/*`
  - `/api/ai/*`

## 4. 开发约束

- 新增端点必须在 PR 描述中给出：
  - 路由前缀
  - 依赖域服务
  - 回归测试清单
- 测试至少包含：
  - 1 个新路径 200 场景
  - 1 个旧路径 404 场景（若存在迁移历史）
