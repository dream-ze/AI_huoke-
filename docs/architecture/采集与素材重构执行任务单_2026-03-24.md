# 采集与素材重构执行任务单（仅保留采集页 + 素材中心）

日期：2026-03-24
适用范围：Desktop 前端、Backend、Browser Extension
目标：
- 保留：数据看板、客户管理、合规审核、线索池、发布任务。
- 重构：采集链路改为“采集直达素材中心”，不再经过素材收件箱。
- 废弃：旧接口和旧页面（收件箱、旧内容接口、旧插件采集接口）全部下线。

---

## 1. 最终业务流（强约束）

手动链接 / 插件一键采集
-> 预提取（可选）
-> 统一入库
-> 素材中心展示（正文块、评论、快照）
-> 异步分析回写洞察
-> AI 改写

约束：
- 不允许再出现“采集 -> 收件箱 -> 素材中心”中间环节。
- 插件只做采集和上报，不做 AI 分析。

---

## 2. 范围与不动项

### 2.1 保留不动（仅回归验证）
- 数据看板
- 客户管理

- 插件：改为统一上报新接口。

## 3. 新接口契约（v2，避免双轨）

> 统一采用 `/api/v2`，上线后旧接口返回 `410 Gone`。

### 3.1 采集预提取
POST `/api/v2/collect/extract-from-url`

请求示例：
```json
{
  "url": "https://www.xiaohongshu.com/explore/xxx"
}
```

响应示例：
```json
{
  "platform": "xiaohongshu",
  "platform_label": "小红书",
  "source_url": "https://...",
  "title": "征信花了还能做吗",
  "content_preview": "最近很多人问我...",
  "author_name": "xxx",
  "metrics": {
    "likes": 123,
    "comments": 45,
    "favorites": 67
  },
  "tags": ["征信", "贷款"],
  "comments_preview": ["查询多还能做吗", "白户可以吗"],
  "fetch_success": true,
  "message": "已完成预提取"
}
```

### 3.2 统一入库
POST `/api/v2/collect/ingest-page`

请求示例：
```json
{
  "source_type": "browser_plugin",
  "client_request_id": "a1f4...",
  "platform": "xiaohongshu",
  "source_url": "https://...",
  "title": "征信花了还能做吗",
  "content_text": "正文全文...",
  "author_name": "xxx",
  "publish_time": "2026-03-23T10:20:00Z",
  "tags": ["征信", "负债"],
  "metrics": {
    "likes": 123,
    "comments": 45,
    "favorites": 67
  },
  "blocks": [
    {"block_type": "paragraph", "block_order": 1, "block_text": "..."},
    {"block_type": "hashtag", "block_order": 2, "block_text": "#征信"}
  ],
  "comments": [
    {"comment_text": "查询多还能做吗", "commenter_name": "用户A", "like_count": 2, "is_pinned": false}
  ],
  "snapshot": {
    "raw_html": "<html>...</html>",
    "screenshot_url": "https://.../shot.png",
    "page_meta_json": {"og:title": "..."}
  },
  "raw_payload": {}
}
```

响应示例：
```json
{
  "content_id": 1024,
  "status": "ingested",
  "dedupe_hit": false,
  "message": "已入素材中心"
}
```

### 3.3 素材列表/详情/更新/删除
- GET `/api/v2/materials`
- GET `/api/v2/materials/{id}`
- PATCH `/api/v2/materials/{id}`
- DELETE `/api/v2/materials/{id}`

### 3.4 异步分析与改写
- POST `/api/v2/materials/{id}/analyze`
- POST `/api/v2/materials/{id}/rewrite`

### 3.5 采集日志
- GET `/api/v2/collect/logs`

## 9. 风险与兜底

- 风险：旧接口调用遗漏导致线上报错。
  - 兜底：发布前通过全局搜索校验旧路径为 0，且在网关层记录 410 命中日志。

- 风险：结构化字段（评论/快照）在部分平台提取为空。
  - 兜底：允许空，但保留 `raw_payload` 和 `raw_html`，后续可回放解析。

- 风险：插件版本未升级。
  - 兜底：后端对旧插件接口返回 410 + 清晰升级提示。
