## 4. 四项风险整改（与你提到的风险逐条对应）

### 风险A：启动依赖 create_all

- 现状：已补齐 Alembic 骨架与迁移链。
- 本阶段动作：
  - 禁止生产继续依赖 create_all 自动改表。
  - 部署脚本统一改为 `alembic upgrade head`。
- 验收标准：生产发布检查项中必须出现 Alembic current/head 对齐。

### 风险B：insight 批量AI分析无速率保护

- 现状：已有 BackgroundTasks，但未加统一限流。
- 本阶段动作：
