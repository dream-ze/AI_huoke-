# 部署后页面与链路验收清单

适用阶段：
- 测试环境部署后
- 正式发布前回归

## A. 系统与运维

1. 打开 `/health`
2. 打开 `/api/system/ops/health`
3. 打开 `/api/system/ops/readiness`
4. 确认 `database`、`redis` 正常
5. 确认容器日志无持续报错

## B. 登录与基础页面

1. 登录页可正常登录
2. 仪表盘可正常加载
3. 页面刷新后登录态不丢失
4. 退出登录后回到登录页

## C. 素材主链路

1. 手动提交一条原始内容到收件箱
2. 确认详情页能看到清洗后的标题与正文
3. 执行 `ingest-and-rewrite`
4. 确认生成结果、知识引用、改写版本正常
5. 采纳一个改写版本，确认素材正文回写

## D. 发布任务闭环

1. 创建发布任务
2. 分配给执行人
3. 执行人领取任务
4. 提交发布结果
5. 确认任务 trace 返回 `lead_id`
6. 重复提交一次，确认线索未重复新增且指标已覆盖更新
7. 驳回任务再重新领取
8. 关闭任务，确认状态流转和反馈记录正常

## E. 线索与客户闭环

1. 打开线索页，确认能看到新线索
2. 修改线索状态
3. 转客户
4. 再次转客户，确认幂等返回同一客户
5. 打开客户页，确认客户详情与来源正常

## F. 权限与隔离

1. 用户 A 生成线索与客户
2. 用户 B 不能查询 A 的 `owner_id`
3. 用户 B 不能改 A 的线索状态
4. 用户 B 不能改派 A 的线索
5. 用户 B 不能追溯 A 的线索
6. 用户 B 不能读取或修改 A 的客户

## G. 异常与回退观察

1. 关闭 Redis 后访问 `/api/system/ops/health`，确认错误信息可见
2. 恢复 Redis 后 `readiness` 恢复为 `200`
3. 查看 backend 日志，确认异常可追踪
4. 如 AI 模型不可达，确认错误能在日志和接口响应中定位
