# 助贷自动获客系统 PRD

版本：V1.0  
日期：2026-04-02  
适用项目：`D:\智获客`

## 1. 文档目的

基于当前“智获客”项目现状，重新定义一个更聚焦业务结果的产品方向：从“AI 内容运营平台”升级为“助贷自动获客系统”。

本文档输出三类内容：

1. 当前项目能力盘点
2. 结合助贷场景的不足与改善建议
3. 面向未来 1-2 个版本的产品需求设计

## 2. 当前项目现状判断

结合当前仓库结构、README、现有接口与模型，项目已经具备以下基础：

- 多端基础骨架：`backend`、`desktop`、`mobile-h5`
- 后端主栈：FastAPI + SQLAlchemy + PostgreSQL + Redis + Ollama/火山方舟
- 已有核心模块：采集、素材、知识库、AI 改写、合规、发布、线索、客户、提醒、会话、看板
- 已有助贷相关字段：`LeadProfile` 中包含贷款金额、房/车/公积金、征信、紧急度等画像信息
- 已有线索评分基础：`lead_scoring_service.py`
- 已有会话建议基础：`conversation.py`

当前项目不算从零开始，已经具备转向助贷获客系统的底座。

但整体判断仍然是：

当前更像“内容采集 + AI 改写 + CRM 雏形”，还不是“围绕助贷转化结果设计的自动获客闭环系统”。

## 3. 当前项目主要不足

## 3.1 核心问题总结

当前最大的不足，不是功能太少，而是功能不少但主链路不够聚焦。

现有链路更像：

采集 -> 素材 -> AI -> 合规 -> 发布 -> 线索 -> 客户

但对助贷行业真正重要的链路应该是：

流量入口 -> 内容触达 -> 私信/留资 -> 线索识别 -> 资质初筛 -> 自动分级 -> 自动跟进 -> 人工接管 -> 成交转化 -> 数据复盘

也就是说，当前系统偏“内容生产”，缺少“获客转化”。

## 3.2 不足明细

### 1. 产品定位不够垂直

当前项目覆盖内容运营、素材管理、知识库、发布、客户、提醒等多个模块，但没有把“助贷获客效率”作为唯一北极星指标。

问题表现：

- 页面多，但核心产出不够清晰
- 功能宽，但针对助贷行业的专属流程不够深
- 模型里已有助贷字段，但业务流程没有围绕这些字段自动运转

建议：

- 将系统定位收敛为“助贷机构/中介的自动获客与线索转化系统”
- 所有模块围绕“线索量、有效线索率、加微率、初筛通过率、成交率”建设

### 2. 线索模型偏记录型，缺少自动决策能力

当前已有 `Lead`、`Customer`、`LeadProfile`，但更多是静态存储。

缺失点：

- 没有完整的“线索来源事件流”
- 没有“留资动作识别”与“多触点归因”
- 没有“自动分配给顾问/团队”
- 没有“按资质与意向自动分层运营”
- 没有“自动淘汰、挂起、再激活”策略

建议：

- 引入线索状态机：新线索、待识别、待初筛、待跟进、跟进中、待补件、已转化、已流失、待激活
- 引入分单策略：按产品线、城市、资质等级、时间段、顾问负载自动派单
- 引入线索事件表：记录每一次触达、咨询、加微、留电、补件、拒绝、转化

### 3. 缺少助贷行业专属获客入口设计

当前采集/发布能力是通用内容平台思路，但助贷获客的入口不只是内容发布，还包括：

- 短视频评论区/私信引流
- 小红书笔记引导
- 微信/企微加粉
- 落地页表单提交
- 渠道投放回传
- 电话/客服/外呼录入

建议：

- 建立统一获客入口中心
- 所有入口统一沉淀到 Lead Intake 层
- 支持内容线索、表单线索、私信线索、企微线索、人工录入线索统一入池

### 4. 自动跟进能力明显不足

从代码看，会话建议与 followup agent 还很初期，未形成真正的自动跟进系统。

问题表现：

- 只能给建议，缺少自动触发
- 没有按线索等级生成不同跟进 SOP
- 没有催办、失联唤醒、补资料提醒的自动化编排
- 没有人工接管条件和风险升级机制

建议：

- 增加“跟进剧本引擎”
- 支持 A/B/C/D 不同线索走不同话术与节奏
- 支持首触达、二次唤醒、资料补充、预约电销、到店邀约等场景模板
- 设置接管阈值：涉及利率、费用、征信修复、投诉、敏感承诺时自动转人工

### 5. 合规能力还不够金融化

当前合规更多偏内容表达审核，对助贷场景仍不够。

助贷场景高风险表达包括：

- 包过、秒批、无视征信、百分百下款
- 利率和费用误导
- 虚假资质包装
- 诱导留资
- 违规承诺和暗示

建议：

- 建立“助贷合规规则库”
- 支持平台规则 + 行业规则双重校验
- 支持文案合规、私聊话术合规、客服回复合规、落地页文案合规
- 输出替代表达，不只拦截

### 6. 数据分析偏运营看板，缺少经营漏斗

目前已有 dashboard 和 funnel 基础，但还不足以支撑助贷经营复盘。

缺失点：

- 线索从曝光到成交的完整漏斗
- 各渠道获客成本对比
- 各顾问接待效率对比
- 各话术模板转化率对比
- 各资质标签人群的成交率对比

建议：

- 看板从“功能数据”改成“经营数据”
- 核心围绕渠道、内容、顾问、产品四个维度分析

### 7. 组织结构偏技术分层，业务域边界还可再收敛

当前后端目录比较完整，但对于 MVP 到 1.0 阶段来说，业务实现可能分散在多个层次，维护成本较高。

建议按业务域重组思路推进：

- acquisition：获客入口与线索接入
- lead_center：线索池、画像、评分、分配、状态机
- followup_center：会话、SOP、提醒、自动跟进、人工接管
- content_growth：选题、文案生成、发布、渠道归因
- compliance_center：文案与话术合规
- analytics_center：经营漏斗与复盘

## 4. 产品目标

## 4.1 产品定位

面向助贷机构、助贷团队、贷款中介、金融咨询团队的自动获客与线索转化平台。

## 4.2 核心目标

帮助运营和顾问团队，实现：

- 更低成本获取线索
- 更高比例识别有效线索
- 更快完成线索分级与分配
- 更规范完成首轮初筛与跟进
- 更高效复盘内容与渠道效果

## 4.3 北极星指标

- 日新增线索数
- 有效线索率
- 加微率/留资率
- 首触达响应时效
- 初筛通过率
- 成交转化率

## 5. 目标用户

### 5.1 主要用户

1. 助贷机构老板/业务负责人  
关注渠道 ROI、顾问产能、成交效率

2. 运营人员  
负责内容选题、账号发布、线索监控、私域承接

3. 电销/顾问  
负责线索跟进、资质初筛、资料收集、成交推进

4. 风控/合规人员  
负责违规表达审核、话术规范、风险监控

## 6. 产品范围

## 6.1 V1 必做范围

1. 获客入口统一接入
2. 线索池与画像体系
3. 自动评分与自动分单
4. 跟进 SOP 与提醒机制
5. 会话建议与人工接管
6. 助贷合规中心
7. 内容-线索-成交归因看板

## 6.2 暂不优先范围

- 复杂 BI 自助分析
- 大规模多组织权限体系
- 完整呼叫中心系统
- 深度营销自动化编排平台
- 多租户 SaaS 计费系统

## 7. 核心业务流程

### 7.1 主链路

1. 运营通过小红书/抖音/知乎/投放页等渠道发布内容或广告
2. 用户通过评论、私信、表单、加微等方式产生线索
3. 系统统一接入线索并完成去重
4. 系统抽取线索画像：贷款金额、资质、征信、城市、紧急程度、产品偏好
5. 系统自动评分并分为 A/B/C/D
6. 系统根据规则自动分配顾问
7. 系统生成首轮跟进建议或自动发送标准回复
8. 遇到敏感问题、投诉或高意向客户时自动转人工
9. 顾问持续推进并更新进展
10. 最终形成成交、流失、待激活等结果
11. 系统回流数据到归因看板，反哺内容与渠道策略

### 7.2 关键状态流转

线索状态：

`new -> identified -> profiled -> scored -> assigned -> contacted -> qualified -> converted / lost / dormant`

会话状态：

`bot_handling -> waiting_user -> risk_detected -> human_takeover -> resumed / closed`

## 8. 功能需求

## 8.1 获客入口中心

### 功能目标

把所有潜在留资入口统一接入，避免线索散落在各平台和人工记录中。

### 需求点

- 内容线索接入：短视频、图文、评论区、私信
- 表单线索接入：H5 表单、落地页表单
- 企微线索接入：加好友、自动欢迎语、标签回写
- 人工录入：电话、地推、转介绍
- 批量导入：Excel、CSV
- 去重策略：手机号、微信号、设备号、昵称相似度、时间窗口

### 输出字段

- 来源渠道
- 来源内容/活动
- 留资方式
- 首次触达时间
- 联系方式
- 平台身份标识

## 8.2 线索池与画像中心

### 功能目标

把线索从“联系人”升级为“可决策的业务对象”。

### 核心字段

- 基础信息：昵称、手机号、微信号、城市
- 需求信息：贷款类型、金额、用途、期限、紧急度
- 资质信息：房/车/公积金/社保/营业执照/征信
- 风险信息：黑名单、敏感表达、投诉记录
- 行为信息：来源渠道、咨询内容、响应次数、最近活跃时间

### 能力要求

- 自动画像抽取
- 手工修正
- 画像置信度展示
- 标签体系：产品标签、资质标签、风险标签、阶段标签

## 8.3 自动评分与分级

### 功能目标

基于线索价值、成交概率和服务优先级自动分层。

### 评分维度

- 需求明确度
- 资质完整度
- 征信风险
- 紧急程度
- 沟通意愿
- 来源可信度

### 分级结果

- A：高意向高资质，优先人工跟进
- B：中高意向，可标准跟进
- C：低意向或资料不全，进入培育池
- D：高风险或低价值，限制资源投入

### 扩展要求

- 支持运营手工调整
- 支持不同产品线不同评分规则
- 支持评分原因解释

## 8.4 自动分配与协同

### 功能目标

把对的线索分给对的人，减少抢单和浪费。

### 需求点

- 按产品类型分配
- 按城市/区域分配
- 按顾问负载分配
- 按班次/工作时间分配
- 支持回收与重分配
- 支持超时未处理自动升级

## 8.5 跟进中心

### 功能目标

把“提醒记一下”升级为“可执行的跟进流程”。

### 核心场景

- 首次触达
- 加微欢迎
- 咨询答疑
- 资料补件提醒
- 征信解释与安抚
- 二次唤醒
- 沉默客户激活
- 流失客户回访

### 需求点

- SOP 模板库
- 不同等级线索不同节奏
- 自动提醒
- 待办列表
- 触达记录沉淀

## 8.6 智能会话与人工接管

### 功能目标

利用 AI 提升回复速度，但关键节点必须可控。

### 需求点

- 会话上下文管理
- AI 回复建议
- 常见问题自动回复
- 敏感意图识别
- 高意向自动标红
- 人工接管机制
- 人工回复沉淀为可复用话术

### 必须接管场景

- 用户询问具体利率、费用、放款时效
- 用户出现投诉、质疑、辱骂
- 用户提及征信修复、包装资料、刷流水
- 用户明确表示近期要贷款且资质较好

## 8.7 助贷合规中心

### 功能目标

降低内容和私聊触达的违规风险。

### 需求点

- 平台内容合规审核
- 私聊话术合规审核
- 留资引导语合规审核
- 敏感词 + 场景语义双校验
- 风险分级：低/中/高
- 替代表达建议
- 审核日志与复盘

## 8.8 内容增长中心

### 功能目标

内容服务于获客，而不是停留在“多写几篇文案”。

### 需求点

- 热点/选题挖掘
- 助贷人群模板：征信花、负债高、个体户、房抵、企业贷等
- 多平台文案生成
- 钩子、标题、评论区引导生成
- 生成内容与实际线索转化绑定

## 8.9 经营分析中心

### 功能目标

让老板、运营、顾问看到真正影响收入的数据。

### 核心看板

1. 渠道看板  
曝光、咨询、加微、留资、有效线索、成交

2. 内容看板  
按平台、账号、内容主题、文案模板分析转化

3. 顾问看板  
响应时效、跟进次数、初筛通过率、成交率

4. 线索质量看板  
A/B/C/D 分布、各标签成交率、各来源有效率

## 9. 页面规划建议

建议将当前页面收敛为以下主导航：

1. 仪表盘
2. 获客入口
3. 线索池
4. 跟进中心
5. 内容增长
6. 合规中心
7. 客户与成交
8. 数据复盘
9. 系统设置

当前项目中的 `materials`、`knowledge`、`publish`、`reminders`、`wecom` 等页面建议逐步纳入以上业务域，不再以技术模块单独暴露。

## 10. 非功能需求

### 性能

- 线索入池接口响应时间 < 2 秒
- 常规线索评分 < 3 秒
- 常规会话建议 < 5 秒

### 稳定性

- 关键链路接口可观测
- 异步任务失败可重试
- 重要动作具备审计日志

### 安全

- 金融类敏感数据脱敏展示
- 顾问权限隔离
- 客户数据访问审计

### 可运营

- 规则可配置
- 标签可配置
- 跟进模板可配置
- 合规词库可更新

## 12. 对当前项目的落地改造建议

## P0：必须先做

1. 重构主导航与主流程，突出“线索池、跟进中心、经营漏斗”
2. 补齐 Lead Intake 统一接入层
3. 完善 `Lead` / `LeadProfile` / `Customer` 之间的状态流转
4. 把线索评分从“算分”升级为“驱动分单和跟进”
5. 把会话建议升级为“自动跟进 + 人工接管”
6. 建立助贷专属合规规则库

## P1：第二阶段做

1. 多触点归因
2. 企微/私域承接
3. 内容转化效果反哺选题
4. 模板级别效果分析

## P2：后续增强

1. 多团队协同
2. 多组织权限
3. 交付版产品化
4. 自动化运营编排

## 13. 成功标准

系统上线后，至少应达到以下效果：

- 线索录入统一率 > 90%
- 自动画像识别覆盖率 > 70%
- 高价值线索 5 分钟内被分配率 > 90%
- 顾问首响时效明显缩短
- 内容到线索的归因可追踪
- 高风险违规表达可提前拦截

## 14. 一句话结论

当前项目已经有了“助贷自动获客系统”的底座，但还缺少真正围绕转化闭环设计的业务主线。

下一步不建议继续横向加页面，而是应该纵向打穿：

`获客入口 -> 线索识别 -> 画像评分 -> 自动分单 -> 自动跟进 -> 人工接管 -> 转化复盘`

这条链路打通后，项目才会从“功能很多的平台”变成“能持续带来线索和成交的系统”。
