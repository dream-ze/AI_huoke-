# 快速任务规划

## 触发条件
- 任务涉及3个以上步骤
- 需要修改多个模块
- 用户明确要求先规划

## 规划模板

### 任务概述
> 一句话描述目标

### 改动清单
按依赖顺序列出：

| 序号 | 文件 | 改动 | 依赖 |
|------|------|------|------|
| 1 | | | - |
| 2 | | | 1 |

### 验证方式
- 如何确认完成？

## 原则
1. **最小改动**：只改必要的
2. **顺序执行**：先改被依赖的
3. **逐步验证**：改一步测一步
4. **回滚预案**：知道怎么撤销

## 复杂度判断
- **简单**（1-2文件）：直接做
- **中等**（3-5文件）：列清单再做
- **复杂**（5+文件）：必须详细规划
